Esta milestone adicionou as funcionalidades core que tornam o app utilizável em produção. Phases 6 (Templates), 12 (Navigation Refactor) foram canceladas. Phase 11 (Notifications) foi adiada para milestone futura.

## Milestone Summary

**Key Decisions:**

- Phase 7 executada com fallback 0 para corrective_actions (Phase 8 ativou a query real)
- canTransitionTo usa createdBy (criador) como avaliador, não "qualquer auditor"
- NotificationService deve ser singleton — necessário para manter unreadCount vivo entre navegações (para Phase 11)
- Upload de imagens é fluxo independente de _saveAnswer — falha de upload não bloqueia finalização de auditoria (core value)
- fl_chart adicionado em Phase 7 e reutilizado em Phase 10 — não adicionado duas vezes
- Escopo de Relatórios substituído por Calendário de Auditorias no Dashboard (Phase 10)
- Phase 6 (Templates) cancelada pelo usuário em 2026-05-02
- Phase 12 (Navigation Refactor) cancelada pelo usuário em 2026-05-02
- Phase 11 (Notifications) adiada para milestone futura

**Issues Resolved:**

- HF-01: Bug dropdown de responsável só mostrava usuário logado (resolvido via quick task 260502)
- HF-02: Excluir tipo de auditoria (resolvido via quick task 260429-q01)
- Bug: parse de deadline como date-only para evitar UTC→local day shift no calendário
- Bug: botão "Alterar responsável" com permissão restrita demais (auditor e responsável agora incluídos)
- Bug: campo "ação tomada" não editável quando status é reaberta (corrigido)
- Novo fluxo de status CAPA: reaberta + cascade cancel ao cancelar auditoria

**Issues Deferred:**

**Technical Debt Incurred:**

- Navegação via drawer permanece (FAB expandível NAV-01/02 cancelado com Phase 12)
