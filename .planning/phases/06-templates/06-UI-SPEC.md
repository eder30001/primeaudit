## Spacing Scale

Declared values (multiples of 4). Source: existing `template_builder_screen.dart` spacing patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dense row gaps (`SizedBox(height: 4)`) |
| sm | 8px | Between list items, between icon and text (`SizedBox(height/width: 8)`) |
| md | 16px | Default element spacing, horizontal list padding (`SizedBox(height/width: 16)`) |
| lg | 24px | Bottom sheet horizontal/vertical padding (`padding: ... 24`) |
| xl | 32px | — (available in scale, not required by this phase) |
| 2xl | 48px | Button height for primary CTAs (48px height on ElevatedButton) |
| 3xl | 64px | — (available in scale, not required by this phase) |

**Exceptions:**
- List bottom padding: 100px (ensures content is not obscured by FAB)
- Section card vertical padding: 10px (acceptable minor deviation for compact section headers)
- Item card leading: `CircleAvatar(radius: 14)` = 28px diameter touch affordance; drag handle icon area extends to 48px via ListTile
- Drag handle hit target: minimum 48px height (Android touch target minimum) — the `ListTile` with `contentPadding: EdgeInsets.symmetric(horizontal: 14, vertical: 4)` meets this when drag handle is a leading widget of the tile

## Typography

Source: `template_builder_screen.dart` existing text styles, normalized to 4 roles.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 (normal) | 1.5 |
| Label | 11px | 400 (normal) | 1.4 |
| Heading | 15px | 700 (bold) | 1.2 |
| Display | 17px | 700 (bold) | 1.2 |

**Notes:**
- Body (13px / w400): item question text, option text, form field content
- Label (11px / w400): subtitle metadata (type, weight, section item count, `Colors.white70` on AppBar)
- Heading (15px / w700): AppBar title (template name), section name, section header
- Display (17px / w700): bottom sheet dialog title ("Novo Item" / "Editar Item")
- Weight 16px/bold is used for numeric weight stepper only — treat as heading role
- Weight w500 and w600 are intermediate weights used inline; contract permits w500 for item card titles and w600 for selected state labels — both round to the "semibold" weight bucket

## Color

Source: `primeaudit/lib/core/app_colors.dart` and `primeaudit/lib/core/app_theme.dart`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F5F7FA` (light) / `#121212` (dark) | Scaffold background, bottom sheet fill, input fill — `AppTheme.background` |
| Secondary (30%) | `#FFFFFF` (light) / `#1E1E1E` (dark) | Item cards, section cards surface — `AppTheme.surface` |
| Accent (10%) | `#1E3A5F` (`AppColors.primary`) | AppBar background, FAB, selected state borders and icons, CircleAvatar fill (10% opacity), ElevatedButton background, section header tint (8% opacity) |
| Action | `#2196F3` (`AppColors.accent`) | Focused input border (2px), `Icons.add_circle_rounded` in option editor |
| Destructive | `#E53935` (`AppColors.error`) | "Excluir" popup menu item, error SnackBar background, error state text |

**Accent reserved for:**
- AppBar background and foreground
- FAB background
- Selected response-type option border (2px) and icon color
- Section header background tint (primary at 8% opacity) and border (20% opacity)
- Section header text, item count, and folder icon
- Item card CircleAvatar background (primary at 10% opacity) and type icon
- ElevatedButton ("Salvar" / "Adicionar") background

**Action (`AppColors.accent` / `#2196F3`) reserved for:**
- Focused state of text input borders (2px accent border replaces divider border on focus)
- "Adicionar opção" icon button

**Destructive (`AppColors.error` / `#E53935`) reserved for:**
- "Excluir seção" and "Excluir item" PopupMenuItem text color
- `TextButton.fromStyle(foregroundColor: AppColors.error)` on confirmation dialogs
- Error SnackBar background

## Component Inventory

### Affected screens

| Screen | File | Change Type |
|--------|------|-------------|
| `AuditExecutionScreen` | `lib/screens/audit_execution_screen.dart` | Bug fix — sort items within section buckets by `orderIndex` after grouping (TMPL-01) |
| `TemplateBuilderScreen` | `lib/screens/templates/template_builder_screen.dart` | Feature — replace flat `ListView` items with `ReorderableListView` per section (TMPL-02) |

### New interaction: drag handle (TMPL-02)

- **Widget:** `ReorderableListView` (built-in Flutter Material, no new package)
- **Gesture:** Long-press on Android/iOS triggers drag; `ReorderableListView` manages the lift animation
- **Drag handle icon:** `Icons.drag_handle_rounded`, size 20, color `AppTheme.textSecondary` — placed as `leading` widget in the `ListTile` replacing the `CircleAvatar`
- **Existing item card:** `_buildItemCard` currently uses `CircleAvatar` as leading. For TMPL-02, wrap each card in `KeyedSubtree(key: ValueKey(item.id))` to satisfy `ReorderableListView` key requirement without changing the `_buildItemCard` signature
- **Drag handle visibility rationale:** `Icons.drag_handle_rounded` is already used in the options list inside `_showItemForm` (line ~196 of builder screen) — consistent with existing affordance. Field auditors on Android tablets need a visible handle cue; long-press alone is not discoverable enough

### TMPL-01: No visual change

The fix to `AuditExecutionScreen._load()` is pure logic — no UI element changes. Items will silently appear in correct order. No new widget, no new animation.

## Interaction States

### Drag-and-drop item reorder (TMPL-02)

| State | Visual |
|-------|--------|
| Idle | Item card with drag handle icon (`Icons.drag_handle_rounded`, `textSecondary` color) on leading |
| Long-press initiated | Flutter `ReorderableListView` default: card lifts with elevation shadow, slight scale increase (platform default) |
| Dragging | Item card floats over list; destination slot shows gap (platform default animation) |
| Drop (success) | Card snaps to new position; local `setState` updates list immediately (optimistic) |
| Drop (persist error) | Error SnackBar (floating, `AppColors.error` background): "Erro ao salvar nova ordem. Revertendo..." then `_load()` restores DB order |
| Loading initial | `CircularProgressIndicator(color: AppColors.primary)` centered on body (existing pattern) |

### TMPL-01 execution screen item order

| State | Visual |
|-------|--------|
| Correct order | Questions appear in `order_index` ascending order within each section — no indicator, this is the expected baseline |

## Accessibility

- **Touch targets:** Drag handle occupies the full `ListTile` leading area. Minimum 48px height enforced by ListTile `contentPadding: EdgeInsets.symmetric(vertical: 4)` on content that is at least 13px text + padding — tiles are taller than 48dp.
- **Semantics:** `ReorderableListView` includes built-in `Semantics` for drag operations on Android (long-press announces "Move item").
- **Color contrast:** `AppColors.primary (#1E3A5F)` on white background passes WCAG AA (contrast ratio ~9.1:1). `AppColors.error (#E53935)` on white passes WCAG AA for large text.
