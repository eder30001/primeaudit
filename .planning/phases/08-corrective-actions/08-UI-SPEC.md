## Spacing Scale

Flutter `EdgeInsets` values (multiples of 4). Matches existing codebase patterns extracted from `audits_screen.dart` and `audit_execution_screen.dart`.

| Token | Value | EdgeInsets Equivalent | Usage |
|-------|-------|-----------------------|-------|
| xs | 4px | `EdgeInsets.all(4)` | Icon-to-label gap, chip internal spacing |
| sm | 8px | `EdgeInsets.all(8)` / `SizedBox(height: 8)` | Vertical gap between form fields, chip row padding |
| md | 16px | `EdgeInsets.all(16)` | Default screen padding, card horizontal padding |
| lg | 24px | `EdgeInsets.all(24)` | Section breaks, dialog padding |
| xl | 32px | `EdgeInsets.all(32)` | Empty-state padding, error padding |
| 2xl | 48px | `SizedBox(height: 48)` | FAB clearance at list bottom |
| 3xl | 64px | Reserved | Not used in this phase |

Exceptions:
- List bottom padding: `EdgeInsets.fromLTRB(16, 12, 16, 100)` — matches `audits_screen.dart` to clear FAB area.
- Filter chip row: `padding: EdgeInsets.only(right: 8)` between chips — matches existing filter pattern.
- Card margin bottom: `EdgeInsets.only(bottom: 8)` — sm token, matches `_AuditCard` in `audits_screen.dart`.
- Touch target for the "create action" icon on execution items: minimum 44×44px (material tap target).

## Typography

Flutter `TextStyle` roles mapped to Material 3 type scale. All sizes in logical pixels.

Allowed weights: `FontWeight.normal` (w400) for body/label roles; `FontWeight.w600` (semiBold) for heading/display roles. No other weights are used in this phase.

| Role | Flutter Style | Size | Weight | Line Height | Usage |
|------|--------------|------|--------|-------------|-------|
| Body | `bodyMedium` | 14px | 400 (normal) | 1.43 (default) | Card secondary text, description field, filter labels |
| Label | `labelMedium` | 12px | 400 (normal) | 1.33 | Status chip labels, item timestamps, helper text, secondary metadata rows (responsible name, due date, linked audit title) — distinguished from Body via smaller size and `t.textSecondary` color only |
| Heading | `titleMedium` | 16px | 600 (semiBold) | 1.5 | Action title in card, form section labels, screen section headers |
| Display | `titleLarge` | 20px | 600 (semiBold) | 1.2 | AppBar title, screen-level headings — distinguished from Heading via larger size only |

Implementation note: declare as `TextStyle(fontSize: N, fontWeight: FontWeight.normal)` or `TextStyle(fontSize: N, fontWeight: FontWeight.w600)` inline — project does not use a centralized `TextTheme` override. Use 12px (labelMedium) for secondary captions.

## RBAC Transition Matrix

Which buttons are shown per role on the detail screen. Never disable and show an error — hide entirely, then show snackbar only if an already-visible (edge case) button is tapped due to state mismatch.

| From Status | To Status | adm | responsavel (is assigned user) | auditor (not assigned) | superuser / dev |
|-------------|-----------|-----|-------------------------------|------------------------|-----------------|
| aberta | em_andamento | show | show | hide | show |
| em_andamento | em_avaliacao | show | show | hide | show |
| em_avaliacao | aprovada | show | hide | show | show |
| em_avaliacao | rejeitada | show | hide | show | show |
| any non-final | cancelada | show | hide | hide | show |
| rejeitada | em_andamento | show | show | hide | show |

"adm" = `AppRole.canAccessAdmin(role)`. "responsavel" = logged-in user UUID matches `action.responsibleUserId`. "auditor" = role is `auditor` and not the responsible. "superuser/dev" = `AppRole.isSuperOrDev(role)`.

## Interaction Contracts

### Form Validation

- Validate on `_formKey.currentState!.validate()` — called only on submit tap, not on change.
- Show validation errors inline below each field (Flutter default `TextFormField` validator behavior).
- Due date past-date check: if selected date is before `DateTime.now()` truncated to day, show validator error "O prazo deve ser uma data futura".

### Navigation Flow

```
HomeScreen (drawer)
  → CorrectiveActionsScreen (list)
      → CorrectiveActionDetailScreen (detail + transitions)
      [back]

AuditExecutionScreen (item card)
  → CreateCorrectiveActionScreen (form)
      [pop on success or cancel]
  → back to AuditExecutionScreen
```

### Error Feedback

All errors: `ScaffoldMessenger.of(context).showSnackBar(SnackBar(content: Text(msg), behavior: SnackBarBehavior.floating))`.

No error widgets in the body for form submission failures — snackbar only. Body error widget (with retry button) only for list/load failures.

### Confirmation Pattern

Destructive transitions (`cancelada`, `rejeitada`) require `showDialog<bool>(AlertDialog(...))` before executing. Pattern matches `_confirmEncerrar` in `audits_screen.dart` exactly: two `TextButton` actions (`Voltar`, then the destructive action in `AppColors.error`).
