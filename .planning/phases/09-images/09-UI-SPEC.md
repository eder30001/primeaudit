## Interaction Contract

### Camera / Gallery Flow

1. User taps camera icon button on a question card.
2. `_PickerSheet` modal appears — user selects "Câmera" or "Galeria".
3. `image_picker` launches native picker. If user cancels, modal dismisses silently — no error, no state change.
4. On image selected: thumbnail appears immediately as `uploading` state using `Image.file` (local path). Upload begins in background via `ImageService.uploadImage()`.
5. On upload success: thumbnail transitions to `uploaded` state using signed URL from Supabase Storage. `setState` on `_ImageStrip`.
6. On upload failure: thumbnail transitions to `error` state. No snackbar shown — error is indicated inline on the tile only. Answer saving is NOT blocked.
7. Retry: user taps the error thumbnail — upload is retried once. If it fails again, error state persists. The user can continue the audit and finalize normally.

### Remove Photo Flow

1. User taps the red `×` badge on an `uploaded` thumbnail.
2. No confirmation dialog — removal is immediate (destructive without confirmation because it is low-stakes during an active audit).
3. `ImageService.deleteImage()` called; thumbnail removed from local list immediately (optimistic removal).
4. If delete fails: thumbnail reappears with error state; inline error icon only — no snackbar.

### Read-Only State

- `readOnly: true` hides the camera icon button entirely.
- Existing uploaded images are shown as thumbnails without the remove badge.
- Tapping a thumbnail opens fullscreen viewer (`showDialog` with `InteractiveViewer` — see Fullscreen Viewer below).

### Fullscreen Image Viewer

- Trigger: tap on any `uploaded` thumbnail (both read-only and active audits).
- Implementation: `showDialog` wrapping `InteractiveViewer` + `Image.network`.
- Background: `Colors.black87`.
- Close: tap outside or `IconButton(Icons.close_rounded, color: Colors.white)` top-right.
- No navigation to a new screen — modal overlay only (satisfies IMG-02 inline requirement).

## Upload State Machine

```
IDLE ──tap──> UPLOADING ──success──> UPLOADED
                        └──failure──> ERROR ──tap──> UPLOADING
UPLOADED ──tap ×──> (removed from list)
```

State is held in `_ImageStrip._uploadStates: Map<String, _UploadState>` where key = local file path (before upload) or storage path (after upload).

Upload failure does NOT affect `_failedSaves` map in `_AuditExecutionScreenState` — the two flows are fully independent (core value: `_saveAnswer` is never blocked by image upload failure).

## Position in `_ItemCard` Widget Tree

The `_ImageStrip` is inserted **after** the observation row and **before** the "Criar ação corretiva" link:

```
_ItemCard.build()
  └── Card
        └── Padding(all: 14)
              └── Column
                    ├── Row (number + question + badges)          [existing]
                    ├── _GuidanceTile (optional)                  [existing]
                    ├── SizedBox(height: 12)                      [existing]
                    ├── _AnswerWidget                             [existing]
                    ├── SizedBox(height: 10)                      [existing]
                    ├── GestureDetector (observation toggle)      [existing]
                    ├── _ImageStrip          ◄─── NEW (Phase 9)
                    │     SizedBox(height: 8) before it
                    ├── GestureDetector (criar ação corretiva)    [existing]
                    └── TextField (observation, if _showObs)      [existing]
```

`_ImageStrip` is always rendered (even when no images exist) in non-read-only mode — it provides the camera button affordance. In read-only mode with no images, it renders `SizedBox.shrink()`.

## States Summary

| State | What the user sees |
|-------|-------------------|
