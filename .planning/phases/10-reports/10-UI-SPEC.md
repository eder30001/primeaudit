## Spacing Scale

Inherits the 8-point scale declared in Phase 7 07-UI-SPEC.md. The following tokens are the complete declared set:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between indicator dots inside a day cell; SizedBox between day-number and dot row |
| sm | 8px | Gap between calendar section title and calendar widget; internal column child gaps |
| md | 16px | Dashboard outer padding (inherited — `EdgeInsets.all(16)`); calendar card internal padding |
| lg | 24px | SizedBox above calendar section (below the two KPI card rows — matches the gap before the chart in the Phase 7 layout plan) |

Exceptions:
- Day cell height: 52px (fixed — accommodates day number + dot row + touch target at comfortable density).
- Day cell width: fills `1/7` of available width via `Table` column widths (no fixed value — adapts to screen).
- Day number area: 32x32px centered circle/pill (tappable days only) — not a spacing token, fixed affordance size.
- Dot indicator diameter: 6px. Dots do not use the spacing scale — they are a fixed visual element.
- Month header height: 40px (icon button row — standard Material `IconButton` default).
- Weekday label row height: 24px (compact label row above the day grid).

Source: Phase 7 07-UI-SPEC.md spacing scale (inherited); calendar-specific values determined during this session.

## Typography

Inherits Phase 7 typography contract. The calendar adds no new sizes or weights beyond what Phase 7 established. Exactly 4 font sizes, exactly 2 font weights across the full dashboard (unchanged from Phase 7):

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 22px | bold (700) | 1.2 | Greeting heading — "Olá, {name}!" (Phase 7, unchanged) |
| KPI value | 20px | bold (700) | 1.0 | KPI card numbers (Phase 7, unchanged) |
| Heading | 15px | bold (700) | 1.2 | Section titles — "Conformidade por template", "Calendário de Auditorias" |
| Body | 13px | regular (400) | 1.5 | KPI card labels; calendar month label; weekday column headers; day cell numbers; empty-state messages |

Calendar-specific typography assignments:
- Month + year label (e.g. "Abril 2026"): 15px bold (700) — Heading role, `textPrimary` color. Centered in month header row.
- Weekday headers (Dom, Seg, Ter, Qua, Qui, Sex, Sáb): 13px regular (400), `textSecondary` color. Centered above each column.
- Day number (tappable day): 13px regular (400), `textPrimary` color. Rendered inside 32x32px circle with `surface` background and `divider` border when it has audits (making the affordance visible). The circle is only drawn when the day has at least one audit.
- Day number (empty day): 13px regular (400), `textSecondary` color. No circle, no decoration. Not tappable.
- Day number (today): 13px bold (700), white color, inside 32x32px filled circle with `AppColors.accent` fill.

Note: Today's date always renders with the accent fill regardless of whether it has audits. If today has audits, the dot row still appears below the circle.

Source: Phase 7 07-UI-SPEC.md typography table (inherited); calendar-specific assignments determined during this session.

**Dot color assignment (D-04):**
- Novas dot = `AppColors.accent` (`#2196F3`) — rascunho + em_andamento statuses
- Atrasadas dot = `AppColors.error` (`#E53935`) — atrasada status
- Concluídas dot = `Colors.green` (`#4CAF50`) — concluida status

**Dot rendering rule:** A dot is rendered only when the count for that status group is > 0. Days with only one status type show one dot; days with all three show three dots in a horizontal row.

## Interaction Contracts

### Month Navigation

- Trigger: user taps left arrow (`Icons.chevron_left_rounded`) or right arrow (`Icons.chevron_right_rounded`).
- Behavior: `_calendarMonth` `DateTime` state field advances or retreats by 1 month. Calendar grid re-renders synchronously from in-memory data — no new network call.
- Past/future limits: none. User can navigate freely to any month.
- On month change: if the displayed month has no audit data loaded, show calendar loading state and trigger a fetch for that month's data. If data for that month is already cached in `_calendarData` map (keyed by `YYYY-MM`), render immediately.
- Performance note: Fetch is month-scoped. Planner should add a `getAuditsForMonth(companyId, year, month)` method to `AuditService` or `DashboardService` rather than fetching all audits again.

### Day Cell Tap (with audits)

- Trigger: user taps a day cell that has at least one audit indicator dot.
- Behavior: `Navigator.push` to `AuditsScreen` with a `DateTime? filterDate` optional parameter (new parameter to add to `AuditsScreen` constructor).
- `AuditsScreen` with `filterDate` set: displays only audits whose effective date (`deadline ?? createdAt`) falls on `filterDate` (same day, month, year — local timezone comparison).
- Filter chip: `AuditsScreen` shows a dismissible filter chip "Auditorias de {DD/MM/YYYY}" above the list when `filterDate` is set. Tapping X on the chip clears the filter and shows all audits.
- Visual affordance: tappable day cells render the day number inside a 32×32px circle (surface background, divider border). The `GestureDetector` wraps the full 52px cell — entire cell area is tappable, not just the circle.

### Day Cell Tap (empty — no audits)

- Behavior: no action. No `GestureDetector` wrapper — cell renders as plain `SizedBox` with text. No visual hover/press state. Blank days from outside the current month also have no tap behavior.
- Rationale: navigating to an empty filtered list adds friction without value. Confirmed by D-06 (Claude decides).

### Pull-to-Refresh

- Trigger: user pulls down at top of scroll (inherited from Phase 7).
- Callback: `_loadDashboard()` — must also reload calendar data for the currently displayed month.
- During refresh: KPI values show `'...'`; calendar shows `CircularProgressIndicator` in a 160px `SizedBox`.
- Behavior: `_calendarMonth` is NOT reset to current month on refresh — user retains their navigation position.

### KPI Loading State

- Unchanged from Phase 7: `_dashboardLoading ? '...' : '$_fieldName'`.
- Calendar loading state shares the same `_dashboardLoading` flag.

### Role-Based Calendar Scoping (D-07)

Calendar data follows exactly the same scoping as KPI cards:

| Role | Calendar data scope |
|------|-------------------|
| auditor | Audits where `auditor_id == currentUser.id` |
| adm | All audits for `activeCompanyId` |
| superuser / dev | All audits for `activeCompanyId` via `CompanyContextService` |

This is enforced in the data layer (service method), not in the widget.

## State Contract

New state fields to add to `_HomeScreenState`:

| State Field | Type | Initial Value | Triggers Re-render |
|-------------|------|---------------|-------------------|
| `_calendarMonth` | `DateTime` | `DateTime(DateTime.now().year, DateTime.now().month)` | yes — drives calendar header label and grid |
| `_calendarData` | `Map<String, List<Audit>>` | `{}` | yes — keyed by `'YYYY-MM-DD'` (ISO date string); value is list of audits for that day |
| `_calendarError` | `String?` | `null` | yes — controls calendar error inline display |

Existing state fields that drive calendar behavior (unchanged):

| Existing Field | How Used by Calendar |
|----------------|---------------------|
| `_dashboardLoading` | Also drives calendar loading spinner — shared flag |
| `_role` | Role scoping for calendar data fetch |

**Day lookup:** To get audits for a specific day cell, use:
```
final key = '${date.year}-${date.month.toString().padLeft(2,'0')}-${date.day.toString().padLeft(2,'0')}';
final dayAudits = _calendarData[key] ?? [];
```

**Effective date assignment logic (D-03):**
When populating `_calendarData`, each audit is keyed by:
```dart
final effectiveDate = audit.deadline ?? audit.createdAt;
if (effectiveDate != null) {
  final key = '${effectiveDate.year}-...-...';
  _calendarData.putIfAbsent(key, () => []).add(audit);
}
```
Audits with both `deadline` and `createdAt` null are silently excluded (not expected in production per D-03).

**Status group computation (D-04):**
From a `List<Audit>` for a day, compute:
```dart
final novas    = audits.where((a) => a.status == AuditStatus.rascunho || a.status == AuditStatus.emAndamento).length;
final atrasadas = audits.where((a) => a.status == AuditStatus.atrasada).length;
final concluidas = audits.where((a) => a.status == AuditStatus.concluida).length;
// cancelada: excluded (D-04)
```

## Layout Structure

Updated `_buildDashboard()` layout with calendar section appended below the KPI rows:

```
_buildDashboard():
  RefreshIndicator (onRefresh: _loadDashboard, color: AppColors.primary)
    SingleChildScrollView (physics: AlwaysScrollableScrollPhysics, padding: EdgeInsets.all(16))
      Column (crossAxisAlignment: start)
        Text greeting (22px bold, textPrimary)
        SizedBox(height: 4)
        Text role-label (13px regular, textSecondary)
        SizedBox(height: 24)
        Row [card Total | card Pendentes] (Expanded each, SizedBox(width: 8) between)
        SizedBox(height: 8)
        Row [card Atrasadas | card Ações abertas OR Empresas]
        SizedBox(height: 24)                              ← NEW: section gap
        Text "Calendário de Auditorias" (15px bold, textPrimary)   ← NEW
        SizedBox(height: 8)                               ← NEW
        _buildCalendar()                                  ← NEW

_buildCalendar():
  if _dashboardLoading:
    SizedBox(height: 160, child: Center(CircularProgressIndicator(color: AppColors.primary)))
  else if _calendarError != null:
    Text _calendarError (13px, textSecondary)
  else:
    Container (surface color, radius: 12, divider border, padding: EdgeInsets.all(16))
      Column
        _CalendarMonthHeader (_calendarMonth, onPrev, onNext)
        SizedBox(height: 8)
        _CalendarWeekdayRow()
        SizedBox(height: 4)
        _CalendarGrid(_calendarMonth, _calendarData, onDayTap)

_CalendarMonthHeader:
  Row (mainAxisAlignment: spaceBetween)
    IconButton (Icons.chevron_left_rounded, color: AppColors.primary, onPressed: _prevMonth)
    Expanded: Text month+year (15px bold, textPrimary, textAlign: center)
    IconButton (Icons.chevron_right_rounded, color: AppColors.primary, onPressed: _nextMonth)

_CalendarWeekdayRow:
  Row: 7× Expanded(Text('Dom/Seg/.../Sáb', 13px, textSecondary, center))

_CalendarGrid:
  Table (7 uniform columns via defaultColumnWidth: FlexColumnWidth())
    → For each week row (rows from first weekday-padded row to last):
      TableRow: 7× _DayCell

_DayCell (has audits, not today):
  GestureDetector (onTap: () => _onDayTap(date))
    SizedBox(height: 52)
      Column (mainAxisAlignment: center, crossAxisAlignment: center)
        Container (32×32, radius: 16, color: surface, border: divider)
          Text(day, 13px, regular, textPrimary, center)
        SizedBox(height: 4)
        _DotRow(novas, atrasadas, concluidas)

_DayCell (today, regardless of audits):
  GestureDetector (onTap: _onDayTap if has audits, else no-op)
    SizedBox(height: 52)
      Column (mainAxisAlignment: center, crossAxisAlignment: center)
        Container (32×32, radius: 16, color: AppColors.accent)
          Text(day, 13px, bold, white, center)
        SizedBox(height: 4)
        _DotRow(novas, atrasadas, concluidas)  ← renders only if any > 0

_DayCell (no audits, not today):
  SizedBox(height: 52)
    Center: Text(day, 13px, regular, textSecondary)
  (no GestureDetector — not tappable)

_DayCell (outside month — padding cell):
  SizedBox(height: 52)  ← blank, no content

_DotRow:
  Row (mainAxisAlignment: center)
    if novas > 0:     Container(6×6, radius: 3, color: AppColors.accent)
    if novas > 0 && (atrasadas > 0 || concluidas > 0): SizedBox(width: 4)
    if atrasadas > 0: Container(6×6, radius: 3, color: AppColors.error)
    if atrasadas > 0 && concluidas > 0: SizedBox(width: 4)
    if concluidas > 0: Container(6×6, radius: 3, color: Colors.green)
  Note: if no dots (empty day with today decoration), render SizedBox(height: 10) as placeholder to maintain cell height consistency
```
