# Phase 13 — UI Design Contract

> Visual and interaction contract for Phase 13: DB Foundation + Template Management.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> This is a Flutter/Material 3 app. All design decisions reference Flutter widgets,
> AppColors, and AppTheme tokens already present in the codebase.

---

## Design System

| Property | Value |
|----------|-------|
| Framework | Flutter + Material 3 (`useMaterial3: true`) |
| Component library | Flutter SDK widgets (no third-party component library) |
| Design token layer | `AppTheme.of(context)` — background, surface, textPrimary, textSecondary, divider |
| Static color constants | `AppColors` — primary, accent, error, background, surface, textPrimary, textSecondary, divider |
| Icon library | Material Icons (built-in Flutter SDK) |
| Font | System default via `ThemeData` (Roboto on Android) |
| Theme modes | Light + Dark via `ValueNotifier<ThemeMode>` + `AppTheme._light` / `AppTheme._dark` |
| Shadcn / Registry | Not applicable — Flutter app |

## Spacing Scale

Declared values (multiples of 4, matching existing codebase patterns):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`SizedBox(width: 4)`), dot row spacing in calendar |
| sm | 8px | Card margin (`margin: EdgeInsets.only(bottom: 10)` ≈ 8), chip horizontal gap |
| md | 16px | Default padding (`padding: EdgeInsets.all(16)`), list tile horizontal padding |
| lg | 24px | Section padding, bottom sheet inner padding (`left: 24, right: 24, top: 24`) |
| xl | 32px | — (used sparingly; prefer lg or 2xl) |
| 2xl | 48px | Drawer header top padding (`fromLTRB(16, 48, 16, 14)`) |
| fab-clear | 100px | ListView bottom padding to clear FAB (`fromLTRB(16, 16, 16, 100)`) |

Exceptions:
- Touch targets: minimum 44×44 logical pixels for all tappable icons (matches existing `width: 44, height: 44` container in `_buildCard`)
- Bottom sheet: always adds `MediaQuery.of(ctx).viewInsets.bottom` to bottom padding (keyboard avoidance)

## Typography

Matches the scale already established across `home_screen.dart`, `audit_templates_screen.dart`, and `corrective_actions_screen.dart`. No new sizes introduced.

| Role | Flutter Size | Weight | Line Height | Usage |
|------|-------------|--------|-------------|-------|
| Display | 22sp | FontWeight.bold (700) | 1.2 | Screen greeting (`Olá, Nome!`), section headings |
| Heading | 17sp | FontWeight.bold (700) | 1.2 | Bottom sheet titles, AppBar subtitles (large) |
| Body | 15sp | FontWeight.w500 (500) | 1.5 | Drawer item labels, primary list tile titles |
| Label | 14sp | FontWeight.w600 (600) | 1.4 | Card template names, tab labels |
| Caption | 12sp | FontWeight.normal (400) | 1.4 | Card descriptions, badge chips, hints |
| Micro | 11sp | FontWeight.normal (400) | 1.3 | Drawer email, drawer role pill, AppBar sub-subtitle |

Rules:
- Body in list tiles: 14–15sp, w500–w600
- Descriptions/secondary text: 12sp, w400, `AppTheme.of(context).textSecondary`
- No font sizes outside this set are introduced in Phase 13

## Color

Sourced directly from `AppColors` and `AppTheme` — no new color values introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `AppTheme.background` — Light: `#F5F7FA` / Dark: `#121212` | Scaffold background, screen background |
| Secondary (30%) | `AppTheme.surface` — Light: `#FFFFFF` / Dark: `#1E1E1E` | Cards, drawer, bottom sheet, input fill |
| Accent (10%) | `AppColors.accent` — `#2196F3` | See reserved-for list below |
| Primary | `AppColors.primary` — `#1E3A5F` | AppBar background, drawer header, FAB background (where type-specific color absent) |
| Destructive | `AppColors.error` — `#E53935` | Delete action in PopupMenuButton, destructive SnackBar, delete confirmation TextButton |
| Divider | `AppTheme.divider` — Light: `#E5E7EB` / Dark: `#3C4043` | Card borders, list dividers, input borders |

Accent (`#2196F3`) reserved for:
- Focused input border (2px, `focusedBorder`)
- FAB icon and label (when no category-specific color is available)
- `RefreshIndicator` spinner color
- `CircularProgressIndicator` color during loading
- Tab indicator underline (`TabBar` indicator color)
- "Global" / "Padrão" chip text color and background tint

Category color tints (from existing codebase pattern — `typeColor.withValues(alpha: 0.12)`):
- Industrial tab: use `Colors.orange` as category tint for icon containers
- Transportadora tab: use `Colors.blue` (distinct from accent) as category tint — `Color(0xFF1565C0)` (deep blue)
- Meus checklists tab: use `AppColors.accent` tint

Status chip colors (established pattern — no new values):
- Padrão badge: `AppColors.accent` tint (`withValues(alpha: 0.1)`), text `AppColors.accent`
- Own/custom badge: `Colors.green` tint, text `Colors.green[700]`

## Screens and Interaction Contracts

### Screen 1: Drawer Entry (NAV-01)

**Widget:** New `_drawerItem` entry in `_HomeScreenState._buildDrawer()`
**Placement:** Between "Auditorias" and "Ações Corretivas" items (above Divider)
**Icon:** `Icons.checklist_rounded`
**Label:** `'Checklist'`
**Visibility:** All roles — no `AppRole` guard (unlike "Templates de Auditoria" which is admin-only)
**Navigation:** Calls `_navigate(ChecklistTemplatesScreen())` — closes drawer, pushes named route

---

### Screen 2: Checklist Templates Screen (TMPLCK-01)

**File:** `lib/screens/checklist/checklist_templates_screen.dart`
**Widget type:** `StatefulWidget` with `TickerProviderStateMixin` (required for `TabController`)

**AppBar:**
- `backgroundColor: AppColors.primary`
- `foregroundColor: Colors.white`
- `title: Text('Checklist', style: TextStyle(fontWeight: FontWeight.bold))`
- `bottom: TabBar(...)` — 3 tabs, indicator color `Colors.white`, label color `Colors.white`, unselectedLabelColor `Colors.white70`

**Tabs (TabBar + TabBarView):**

| Index | Label | Icon | Content |
|-------|-------|------|---------|
| 0 | `'Industrial'` | `Icons.factory_outlined` | Templates with `category == 'industrial'` (seeds + own) |
| 1 | `'Transportadora'` | `Icons.local_shipping_outlined` | Templates with `category == 'transportadora'` (seeds + own) |
| 2 | `'Meus checklists'` | `Icons.person_outline_rounded` | Templates created by current user (`created_by == currentUserId`) |

**FAB:** `FloatingActionButton.extended`
- `icon: Icons.add`
- `label: Text('Novo checklist')`
- `backgroundColor: AppColors.primary`
- `foregroundColor: Colors.white`
- Visible on all tabs (creation always lands in "Meus checklists" regardless of tab)

**Loading state:** `Center(child: CircularProgressIndicator(color: AppColors.primary))`

**Empty state — Industrial / Transportadora tabs:**
- Icon: `Icon(Icons.factory_outlined` / `Icons.local_shipping_outlined`, size: 48, color: `AppTheme.textSecondary`)`
- Heading: `'Nenhum template disponível'` — 16sp, w500, `textSecondary`
- Body: `'Os templates padrão serão carregados em breve.'` — 13sp, `textSecondary`

**Empty state — Meus checklists tab:**
- Icon: `Icon(Icons.checklist_rounded, size: 48, color: AppTheme.textSecondary)`
- Heading: `'Nenhum checklist criado'` — 16sp, w500, `textSecondary`
- Body: `'Crie um checklist personalizado ou clone um template existente.'` — 13sp, `textSecondary`

**Error state:**
- Inline text: `'Erro ao carregar templates. Puxe para atualizar.'` — 13sp, `textSecondary`
- `RefreshIndicator` wraps each tab's `ListView` (pull-to-refresh calls `_load()`)

### Screen 2a: Template Card (within each tab)

**Widget:** Private `_ChecklistTemplateCard` (reusable across all 3 tabs)
**Outer:** `Card` — elevation: 0, borderRadius: 12, border: `BorderSide(color: AppTheme.divider)`
**Card margin:** `EdgeInsets.only(bottom: 10)`

**Layout:**
```
[Icon container 44×44] | [Name w600 14sp + description 12sp + badges row] | [trailing]
```

**Icon container:**
- 44×44, borderRadius: 10
- Background: `categoryColor.withValues(alpha: 0.12)`
- Icon: `Icons.checklist_rounded` (all templates) or category-specific icon

**Badges row (below description):**
- "Padrão" chip: shown when `is_padrao == true` — accent tint bg, accent text, 11sp w600, px: 8, py: 2
- "Personalizado" chip: shown when `is_padrao == false && created_by == currentUserId` — green tint bg, green[700] text

**Trailing:**
- For seed templates (`is_padrao == true`): `Icon(Icons.copy_outlined, color: textSecondary, size: 20)` — tapping opens clone confirmation bottom sheet
- For own templates (`created_by == currentUserId`): `PopupMenuButton` with options: Editar, Clonar, Excluir
- For other users' templates (edge case, not expected in v1.2 scope): `Icon(Icons.chevron_right_rounded, color: textSecondary)`

**Tap behavior:** Seeds → triggers clone flow; own templates → opens `ChecklistTemplateFormScreen` in edit mode

### Screen 3: Create / Edit Template Form (TMPLCK-02, TMPLCK-03)

**File:** `lib/screens/checklist/checklist_template_form_screen.dart`
**Widget type:** `StatefulWidget` (full screen, not bottom sheet — items list can be long)
**Mode:** Passed via `final ChecklistTemplate? editing` parameter. `null` = create mode.

**AppBar:**
- Create mode: `title: Text('Novo checklist')`
- Edit mode: `title: Text('Editar checklist')`
- `backgroundColor: AppColors.primary`, `foregroundColor: Colors.white`
- No save button in AppBar — save is via primary CTA button at bottom

**Form fields (top to bottom, padding: EdgeInsets.all(16)):**

1. **Nome** — `TextFormField`
   - Label: `'Nome do checklist *'`
   - Prefix icon: `Icons.assignment_outlined`
   - Validation: required, non-empty
   - `textCapitalization: TextCapitalization.words`

2. **Categoria** — `DropdownButtonFormField<String>`
   - Label: `'Categoria *'`
   - Prefix icon: `Icons.category_outlined`
   - Options: `'industrial'` → `'Industrial'`, `'transportadora'` → `'Transportadora'`
   - Validation: required

3. **Descrição** — `TextFormField`
   - Label: `'Descrição (opcional)'`
   - Prefix icon: `Icons.notes_rounded`
   - `maxLines: 3`
   - No validation

4. **Itens section heading:**
   - `Text('Itens do checklist', style: TextStyle(fontSize: 16, fontWeight: FontWeight.w600))`
   - `SizedBox(height: 8)`

5. **Items list** — `ListView` of `_ItemRow` widgets (each item is a TextFormField row with a remove button)
   - Each row: `[TextField — item description] [IconButton(Icons.remove_circle_outline, color: error)]`
   - Minimum 1 item required (form validator checks `_items.isNotEmpty`)

6. **Add item button:**
   - `TextButton.icon(icon: Icon(Icons.add), label: Text('Adicionar item'))`
   - Color: `AppColors.accent`

7. **Primary CTA — full-width ElevatedButton:**
   - Create mode: `'Criar checklist'`
   - Edit mode: `'Salvar alterações'`
   - `backgroundColor: AppColors.primary`, `foregroundColor: Colors.white`
   - Height: 48, borderRadius: 12, elevation: 0
   - Shows `CircularProgressIndicator(color: Colors.white, strokeWidth: 2)` during save

**Input decoration (applied to all fields) — matches existing `_inputDec` pattern:**
- `filled: true`, `fillColor: AppTheme.surface`
- `border / enabledBorder`: `OutlineInputBorder(borderRadius: 12, color: divider)`
- `focusedBorder`: `OutlineInputBorder(borderRadius: 12, color: accent, width: 2)`
- `contentPadding: EdgeInsets.symmetric(horizontal: 16, vertical: 14)`
- `prefixIcon` size: 20, color: `AppTheme.textSecondary`

**Spacing between fields:** `SizedBox(height: 14)` (matches audit_templates_screen.dart)

### Screen 4: Clone Template Flow (TMPLCK-05)

**Trigger:** Tapping a seed card OR selecting "Clonar" from own template's `PopupMenuButton`

**Implementation:** `showModalBottomSheet` (not a full screen — clone is a fast operation)

**Bottom sheet content:**
- Title row: `Icon(Icons.copy_outlined)` + `Text('Clonar template')` — 17sp bold
- Body text: `'Clonar "${templateName}" para Meus checklists?'` — 14sp, `textSecondary`
- `SizedBox(height: 4)`
- Sub-body: `'Você poderá editar os itens depois.'` — 13sp, `textSecondary`
- `SizedBox(height: 24)`
- CTA button: `ElevatedButton` full-width — `'Clonar checklist'`
  - `backgroundColor: AppColors.primary`, `foregroundColor: Colors.white`
  - Height: 48, borderRadius: 12, elevation: 0
  - During clone: shows `CircularProgressIndicator(color: Colors.white, strokeWidth: 2)` inline (button content replaced, button disabled)
- Cancel: `TextButton` — `'Cancelar'` — `AppTheme.textSecondary` color

**Post-clone feedback:** `SnackBar(behavior: floating)` — `'Checklist clonado com sucesso. Acesse "Meus checklists".'`

**Error:** `SnackBar(behavior: floating, backgroundColor: AppColors.error)` — `'Erro ao clonar. Tente novamente.'`

---

### Interaction: Delete Template (TMPLCK-04)

**Trigger:** "Excluir" in `PopupMenuButton` trailing of own template card

**Confirmation:** `showDialog<bool>` — `AlertDialog`
- `title: Text('Excluir checklist')`
- `content: Text('Excluir "${templateName}"? Todos os itens serão removidos e essa ação não pode ser desfeita.')`
- Actions:
  - `TextButton('Cancelar')` — default style
  - `TextButton('Excluir', style: TextButton.styleFrom(foregroundColor: AppColors.error))`

**Post-delete feedback:** `SnackBar(behavior: floating)` — `'Checklist excluído.'`
**Error:** `SnackBar(behavior: floating, backgroundColor: AppColors.error)` — `'Erro ao excluir. Tente novamente.'`

**Guard:** Seeds (`is_padrao == true`) must never show the delete option. The `PopupMenuButton` trailing is only rendered when `created_by == currentUserId`. This is enforced in the widget, not just via RLS.
